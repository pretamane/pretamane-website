üìã How to Test Cache Behavior

  1. Open Chrome DevTools: Press Ctrl+Shift+I (or F12)
  2. Go to Network Tab: Click on "Network" in DevTools
  3. Enable Preserve Log: Check "Preserve log" checkbox
  4. Clear Network Log: Click the üö´ icon to clear existing entries
  5. First Load (Normal Reload):
    • Press Ctrl+R (normal reload)
    • Look at Network tab - check status codes
    • Status 200 = Fresh load
    • Status 304 = Served from cache (Not Modified)
    • Size column shows (disk cache) or (memory cache) = Cached
  6. Hard Reload Test:
    • Press Ctrl+Shift+R (hard reload)
    • Look at Network tab again
    • All resources should show 200 status (not 304)
    • Size column should show actual file sizes (not cache)
    • Request Headers should include Cache-Control: no-cache
  7. Verify in Headers:
    • Click on any resource in Network tab
    • Go to "Headers" tab
    • Look at "Request Headers" section
    • Hard reload should show: Cache-Control: no-cache and Pragma: no-cache

What to Look For

Reload Type Status Code Size Column Request Headers
Ctrl+R (Normal) 200 or 304 (disk cache) or (memory cache) Normal headers, may include If-Modified-Since
Ctrl+Shift+R (Hard) 200 (always) Actual file size (KB) Cache-Control: no-cache
Pragma: no-cache

Console Commands for Verification

Open Console tab in DevTools and run these commands:

performance.getEntriesByType('resource').filter(r => r.name.includes('test-cache-reload')).map(r => ({name: r.name, transferSize: r.transferSize, decodedBodySize: r.decodedBodySize, fromCache: r.transferSize === 0}))

Explanation:

  • transferSize === 0 = Served from cache
  • transferSize > 0 = Downloaded from server (cache bypassed)
